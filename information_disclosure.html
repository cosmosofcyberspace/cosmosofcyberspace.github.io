Information disclosure

1.) Information disclosure in error messages

Bu lab’da requestte bulunan herhangi bir parametreye beklenenin dışında bir değer verdiğinde basılan hata mesajı ile bilgi ifşasını ele almaktadır.

Aşağıdaki productId parametresine verilen beklenmeyen bir değer ile Responsedaki hata mesajında versiyon bilgisi ifşası elde edilmektedir.

2.) Lab: Information disclosure on debug page

Bu lab’da phpinfo.php sayfasından elde edilebilen bilgilerin ifşası söz konusudur.

Lab’da /cgi-bin/phpinfo.php ‘ de bir SECRET_KEY’in bulunabileceği vurgulanmıştır.

3.) Source code disclosure via backup files

Bu labda robots.txt’de bulunan /backup dizinden , saldırgana bilgi ifşası çıkartacak , backup dizininde bulunan bir dosyaya gideceğiz.

Peki robots.txt nedir ?

Robots.txt ; web uygulamalarında hassas olarak görülen dizinlerin web tarayıcıları tarafından ya da dizin tarama araçları tarafından taranmasının önüne geçmek için geliştirilmiş bir tekniktir.Fakat bu ekstra güvenliğin aksine saldırgana “burada da ne varmış” etkisi yaratacak bir dosya olduğundan güvenliğin aksine zafiyet yaratır.

Bahsettiğimiz etkiyi yaşar ve /backup dizinini ziyaret ederiz.

href attribute’unda bir adres görür ve oraya gidersek ; /backup/ProductTemplate.java.bak

Birçok kritik bilginin ifşasını elde etmiş oluruz.

4.) Authentication bypass via information disclosure

“Admin interface only available to local users“

Bu tarz bir yazı ile local bir user olma şartı görüldüğünde aklınıza X-Custom-IP-Authorization ile bypass gelebilir.

X-Custom-IP-Authorization header’ına local ip yazmak düşünülebilir (örneğin 127.0.0.1 ) çünkü sadece local user’lar için erişime açık ! ve burada bu header a yazacağımız local ip değeri ile authenticated request yapabiliriz.

Response'tan anlaşılabilecek olanlar ;

admin paneline sadece 2 durumda giriş yapılabilir ;

  • admin kullanıcısı olarak authenticated olman lazım
  • X-Custom-IP-Authorization header'ı local IP olmalı

Not :

X-Custom-IP-Authorization header’ın kulanım amacı ; Bir isteğin local ana bilgisayar IP adresinden gelip gelmediğini belirlemek için kullanılır.

Not:

TRACE metodunun açık olduğu web uygulamasında , HttpOnly flag’i açık olsa bile TRACE metodundan dolayı saldırgan cookieleri çalabilir , bu saldırının adı Cross-Site Tracing’dir.
